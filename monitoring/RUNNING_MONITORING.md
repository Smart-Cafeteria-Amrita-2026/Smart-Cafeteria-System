# How to Run and Test Monitoring

This guide explains how to start the monitoring stack (Prometheus, Loki, Grafana) and generate test traffic for verification.

## Prerequisites
- Docker and Docker Compose installed.
- PowerShell (for running the traffic script).

## 1. Start the Monitoring Stack
Navigate to the `monitoring` directory and run:

```bash
cd monitoring
docker-compose up -d --build
```

This starts the following services:
- **Backend App**: `http://localhost:3001`
- **Prometheus**: `http://localhost:9090`
- **Loki**: `http://localhost:3100`
- **Grafana**: `http://localhost:3002` (Login: `admin` / `admin` or skipped if anonymous)
- **Promtail**: Log collector (background service)

## 2. Verify Services
Check if all containers are running:
```bash
docker ps
```
You should see `smc_backend`, `smc_prometheus`, `smc_loki`, `smc_grafana`, and `smc_promtail`.

## 3. Generate Test Traffic
To test if logs and metrics are being collected, run the traffic generation script. This script simulates real user traffic including successful requests and errors (which show up in logs).

On Windows (PowerShell):
# If you are in the project root:
.\monitoring\continuous_traffic.ps1

# OR if you are inside the 'monitoring' directory:
.\continuous_traffic.ps1

Keep this script running in a separate terminal window. You will see output like:
- `GET /api/auth/test: Success` (200 OK)
- `POST /api/auth/signIn: 400 Bad Request` (Expected Error)
- `GET /api/random...: 404 Not Found` (Expected Error)

## 4. View Logs and Metrics
1. Open **Grafana** at [http://localhost:3002](http://localhost:3002).
2. Go to **Explore** (compass icon).
3. Select **Loki** as the data source.
4. Run a query like `{job="docker"}` to see all logs, or filter by `{container_name="smc_backend"}`.
5. You should see the traffic generated by the script appearing in real-time.

## 5. Stop Monitoring
To stop all services:
```bash
docker-compose down
```
